### 一.说出软件和应用程序的区别?

核心结论：**应用程序是软件的子集，软件包含应用程序**，范围和用途不同：

- 软件（Software）：广义上指计算机中所有非硬件的部分，是 “指令和数据的集合”，涵盖范围极广，分为两大类：

  1. 系统软件（比如 Windows 系统、macOS、Linux、驱动程序）：负责管理硬件、支撑其他软件运行，是 “电脑的底层基础”；
  2. 应用软件（Application）：为满足特定用户需求而开发的软件，是 “基于系统软件运行的工具 / 程序”。

- 通俗理解：

  软件 = 系统软件 + 应用软件

  比如：电脑里的 Windows 11 是「系统软件」，而 Typora、浏览器、微信是「应用程序」；手机里的 iOS 是「系统软件」，抖音、小红书是「应用程序」。

### 三.整理出网页从编写到浏览器显示的整个过程

1. 编写阶段：开发者用 HTML（结构）、CSS（样式）、JS（交互）编写网页文件（.html、.css、.js）；
2. 部署阶段：将编写好的文件上传到「服务器」（具备公网 IP 的计算机），服务器开启 Web 服务（比如 Nginx）；
3. 访问阶段：用户在浏览器输入域名（比如[www.baidu.com](https://www.baidu.com/)），浏览器通过 DNS 解析（把域名转换成服务器 IP 地址），向服务器发送 HTTP 请求；
4. 响应阶段：服务器接收请求后，找到对应的网页文件，将文件内容（HTML/CSS/JS 等数据）通过 HTTP 协议返回给浏览器；
5. 解析渲染阶段：
   - 浏览器先解析 HTML，生成 DOM 树（网页的结构骨架）；
   - 再解析 CSS，生成 CSSOM 树（网页的样式规则）；
   - 结合 DOM 树和 CSSOM 树，生成渲染树（最终要显示的内容）；
   - 浏览器根据渲染树绘制页面（布局→绘制→合成），最终显示出完整网页。

### 四、服务器是什么？说出你的理解

- 本质：**一台具备公网 IP、高性能、24 小时开机的计算机**（硬件层面），同时安装了 Web 服务软件（比如 Nginx、Apache）和数据库（比如 MySQL）（软件层面）。
- 核心作用：存储网页 / 应用程序的文件（HTML/CSS/JS、后端代码、数据），并响应用户的访问请求（比如你打开百度，就是服务器把百度的网页文件发送给你的浏览器）。
- 通俗理解：服务器是 “网页的存放仓库 + 数据中转站”，就像一家商店的 “仓库 + 店员”—— 仓库存放商品（网页文件），店员（Web 服务）接收顾客（浏览器）的需求，把商品递交给顾客。

### 五、网页的三大组成部分是哪些？分别说出他们的作用

网页的三大核心支柱：HTML、CSS、JavaScript，分工明确、缺一不可：

1. HTML（超文本标记语言）：

   

   作用：定义网页的「结构」（骨架），比如标题、段落、图片、按钮等元素的位置和层级。

   

   举例：用

   ```
   <h1>
   ```

   定义一级标题，

   ```
   <p>
   ```

   定义段落，

   ```
   <img>
   ```

   插入图片。

2. CSS（层叠样式表）：

   

   作用：控制网页的「样式」（外观），比如颜色、字体、大小、布局、间距等。

   

   举例：用

   ```
   color: red
   ```

   把文字改成红色，

   ```
   font-size: 18px
   ```

   设置字体大小，

   ```
   margin: 10px
   ```

   设置间距。

3. JavaScript（脚本语言）：

   

   作用：实现网页的「交互功能」（动态效果），比如点击按钮弹出弹窗、表单验证、数据加载、轮播图切换等。

   

   举例：点击 “提交” 按钮后，检查输入内容是否合法；滚动页面时，导航栏固定在顶部。





### 六.浏览器的内核是什么?有哪些常见内核?

- 浏览器内核（又称渲染引擎）：**浏览器的核心组件，负责解析 HTML、CSS、JS，并将其渲染成可视化的网页**，决定了浏览器的兼容性、渲染速度和功能支持。
- 常见内核（主流 + 非主流）：
  1. Blink（ blink ）：
     - 所属浏览器：Chrome（谷歌）、Edge（新版）、Opera（新版）、Brave；
     - 特点：开源、渲染速度快、支持最新 Web 标准，是目前最主流的内核。
  2. Gecko（ 壁虎 ）：
     - 所属浏览器：Firefox（火狐）；
     - 特点：开源、兼容性强、注重隐私保护，对开发者友好。
  3. WebKit（ 韦伯 kit ）：
     - 所属浏览器：Safari（苹果）、老版 Chrome/Opera；
     - 特点：轻量、渲染流畅，专为苹果设备优化（Mac/iOS）。
  4. Trident（ 三叉戟 ）：
     - 所属浏览器：IE 浏览器（已淘汰）；
     - 特点：兼容性差、不支持现代 Web 标准，目前基本不再使用。

### 八.元素结构是什么?有哪些单标签元素,双?

#### 1. 元素结构（HTML 元素 = 标签 + 属性 + 内容）

HTML 元素是构成网页的基本单位，核心结构：

- 双标签元素（闭合标签）：`<标签名 属性="值"> 内容 </标签名>`

  举例：

  ```html
  <p class="text"> 这是段落 </p>
  ```

  ```html
  <p>
  ```

  是开始标签，

  ```html
  class="text"
  ```

  是属性，“这是段落” 是内容，

  ```html
  </p>
  ```

  是结束标签）；

- 单标签元素（自闭合标签）：`<标签名 属性="值">` 或 `<标签名 属性="值" />`（无需结束标签，自身完成功能）。

#### 2. 常见单标签元素（自闭合，无内容）：

`<br>`（换行）、`<img>`（插入图片）、`<input>`（输入框）、`<meta>`（元信息）、`<link>`（引入外部文件，如 CSS）、`<hr>`（水平线）。

#### 3. 常见双标签元素（需闭合，有内容）：

`<html>`（根元素）、`<head>`（头部）、`<body>`（身体）、`<h1>-<h6>`（标题）、`<p>`（段落）、`<div>`（容器）、`<span>`（行内容器）、`<a>`（超链接）、`<ul>`（无序列表）、`<li>`（列表项）、`<table>`（表格）、`<form>`（表单）。

### 九.元素之间有哪些关系?写出一个例子,并且说明他们的关系

#### 1. 元素关系分类：

- 父元素与子元素：A 元素包含 B 元素，A 是父，B 是子（直接包含）；
- 祖先元素与后代元素：A 元素包含 B 元素，B 包含 C 元素，A 是 C 的祖先（间接包含）；
- 兄弟元素：同一父元素下的多个元素（比如同一`<div>`里的两个`<p>`）；
- 根元素：整个 HTML 文档的最外层元素（只有`<html>`是根元素）。

#### 2. 例子（结合关系说明）：

```html
<html> <!-- 根元素，所有元素的祖先 -->
  <body> <!-- <html>的子元素；<div>、<p>的父元素/祖先 -->
    <div class="container"> <!-- <body>的子元素；<h2>、<ul>的父元素 -->
      <h2>我的笔记</h2> <!-- <div>的子元素；与<ul>是兄弟元素 -->
      <ul> <!-- <div>的子元素；与<h2>是兄弟元素；<li>的父元素 -->
        <li>HTML基础</li> <!-- <ul>的子元素；与下一个<li>是兄弟元素 -->
        <li>CSS样式</li> <!-- <ul>的子元素；与上一个<li>是兄弟元素 -->
      </ul>
    </div>
    <p>这是一个独立段落</p> <!-- <body>的子元素；与<div>是兄弟元素 -->
  </body>
</html>
```

#### 关系总结：

- `<body>` 是 `<div>` 和 `<p>` 的父元素，`<div>` 是 `<h2>` 和 `<ul>` 的父元素；
- `<h2>` 和 `<ul>` 是兄弟元素，两个 `<li>` 是兄弟元素；
- `<html>` 是所有元素的祖先元素，`<li>` 是 `<div>`、`<body>`、`<html>` 的后代元素。

### 十.注释的作用,html的注释如何编写

#### 1. 注释的作用：

- 对代码进行说明（方便自己或他人理解代码功能）；
- 临时注释掉不需要运行的代码（调试时用，浏览器不会渲染注释内容）；
- 注释内容不会显示在网页上，仅在源代码中可见。

#### 2. HTML 注释的编写格式：

- 语法：`<!-- 注释内容 -->`（开头`<!--`，结尾`-->`，中间是注释内容）；
- 注意：注释不能嵌套（比如`<!-- 外层注释 <!-- 内层注释 --> -->`是错误的）；
- 示例：

```html
<!-- 头部区域：存放页面元信息 -->
<head>
  <meta charset="UTF-8"> <!-- 字符编码：UTF-8（支持中文） -->
  <title>笔记页面</title>
</head>
<body>
  <h1>HTML学习</h1>
  <!-- <p>这是一段被临时注释的文本，浏览器不会显示</p> -->
</body>
```
